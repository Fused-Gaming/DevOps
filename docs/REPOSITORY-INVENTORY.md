## ğŸ“‹ Version Tracking Strategy

### Current State
- **Only DevOps has formal releases** using semantic versioning (v2.3.0)
- Other repositories use package.json versions but no GitHub releases
- No centralized version tracking across projects

### Recommendations

#### 1. **Adopt Semantic Versioning for All Projects**
```
MAJOR.MINOR.PATCH
- MAJOR: Breaking changes
- MINOR: New features (backward compatible)
- PATCH: Bug fixes
```

#### 2. **Create GitHub Releases**
- Use `gh release create` for each milestone
- Tag releases with `v` prefix (e.g., v1.0.0)
- Generate release notes from CHANGELOG.md

#### 3. **Automated Release Workflow**
Create `.github/workflows/release.yml` in each repository:
```yaml
name: Create Release
on:
  push:
    tags:
      - 'v*'
jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Create Release
        run: gh release create ${{ github.ref_name }} --generate-notes
```

#### 4. **Version Synchronization**
- Update package.json version before tagging
- Keep CHANGELOG.md in sync
- Use `scripts/update-version.sh` for atomic updates

#### 5. **Cross-Repository Dependencies**
Track version dependencies between projects:
```json
{
  "dependencies": {
    "vln-design-system": "^0.1.0",
    "wallet-sdk": "^1.0.0"
  }
}
```
